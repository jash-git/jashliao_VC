微軟Visual Studio2005開發工具路線圖詳解
出處：MSDN[ 2005-11-11 11:32:04 ] 作者：MSDN 責任編輯：xietaoming

Visual C++

　　Visual C++ 2005 將為面向 Windows 和 .NET Framework 的系統和應用程式開發人員提供比其較低版本更多的功能和更細粒度的控制。 人們計劃在該產品的所有領域（包括語言、編譯器和核心庫）進行關鍵改進。這些增強功能將使 C++ 開發人員能夠更有效地面對 CLR、優化代碼的性能、面對 64 位 Windows 以及提高應用程式的安全性。此外，Visual C++ 2005 將 使開發人員能夠直接在 Visual Studio 2005 IDE 中為移動設備產生本機 C++ 應用程式。（有關嵌入式 C++ 開發的詳細資訊，請參閱 “基於設備的開發”一節）。

　　透過 Visual C++ .NET 2002 版和 2003 版，託管擴展使開發人員能夠顯式利用 CLR 和 .NET Framework。在 Visual C++ 2005 中，現有的託管擴展語法將繼續得到完全支援。此外，Visual C++ 將透過強大的新功能以及用於編寫託管代碼的更為優雅的語法為 CLR 和 .NET Framework 提供擴展支援。該語法現正在作為到 ECMA 標準化公共語言基礎結構 (CLI) 的 C++ 綁定而進行 ECMA 標準化。這一新語法將稱為 C++/CLI。

　　C++ 語言的 C++/CLI 子集主要將在 Visual C++ 2005 中實現，並將為 Visual C++ 開發人員提供很多新功能。C++ 開發人員將第一次能夠使用真正的 C++ 程式設計范型編寫 .NET 連接的應用程式。例如，將為由 CLR 和垃圾回收器管理的代碼和數據啟用 C++ 模板和 C++ 樣式的確定性終止。此外，C++/CLI 還為 C++ 提供很多 CLR 程式設計范型，包括泛型的 CLR 實現。

　　C++/CLI 還簡化了定義和使用託管數據類型的方式。新標準引入的關鍵字對於 C++ 開發人員而言，比目前託管擴展的下劃線-下劃線 關鍵字更為直觀。在 C++/CLI 中，ref 關鍵字用來以與託管擴展 __gc 關鍵字類似的方式定義垃圾回收類：

ref class MyRefClass { /* ... */ };

　　此外，C++/CLI 還引入一種新的類型 － 句柄，該類型用來表示自動垃圾回收的使用。句柄借用指針的語法，但使用加字元號 (^) 而不是星號 (*)。關鍵字 gcnew 用於創建上述垃圾回收對象，並且返回一個句柄：

MyRefClass ^ c = gcnew MyRefClass();

　　透過 C++/CLI，開發人員還將能為句柄類型合併入傳統的 C++ 樣式確定性終止。這意味著可使用正常的 C++ 堆疊分配語法選擇性地將引用類型作為對象進行實例化。這樣，該對象將在脫離作用範圍時自動調用它的析構函數。該功能啟用了在其他面向 CLR 的語言中不可用的程式設計風格。

　　除了 C++/CLI 支援以外，Visual C++ 編譯器還將為面向 CLR 的代碼提供多個代碼產生增強功能。這些增強功能包括：能夠指定不包含本機 x86 指令的“純粹”圖像，以及透過 CLR 驗證過程的“安全”圖像。而且，Visual C++ 2005 編譯器能夠比其他託管語言更好地優化 MSIL。

　　為 Visual C++ 2005 中的 C++ 編譯器規劃了很多增強功能，其中之一便是 Profile Guided Optimizations (POGO)。POGO 技術使編譯器能夠測量應用程式的各種數據，並收集有關應用程式在運行時使用方式的資訊。這些資訊使 Visual C++ 能夠基於真實的應用程式使用模式進一步優化產生的代碼。目前，在免費下載的 Platform SDK (PSDK) 中附帶有 64 位 POGO 技術的預發佈版本。在 Visual C++ 2005 中，該技術將擴展到核心 32 位編譯器，且有關的支援將集成到 IDE 中（圖 7）。


圖 7. Visual C++ 2005 中的 POGO 支援將透過規范和運行時分析來優化應用程式。

該版本還將包含目前在 PSDK 中僅以預發佈形式提供的 64 位 C++ 編譯器。這些編譯器使 Visual C++ 開發人員能夠編寫面向在 Intel 和 AMD 硬體上運行的 64 位版本 Windows 的非託管代碼。對面向 64 位 Windows 的支援將完全集成到 IDE 中（圖 8）。


圖 8. Visual C++ 2005 中的平台支援範圍被擴展以包括 64 位 Windows 以及各種移動設備。

　　除語言和編譯器增強功能以外，Visual C++ 2005 還為核心 C++ 庫（如 Microsoft 基礎類庫 (MFC)、標準模板庫 (STL) 和 C 運行時庫 (CRT)）提供改進和增強。這些增強將使 C++ 開發人員能更好地支援 .NET Framework，並提供內置的應用程式安全性。

　　首先，將引入新版本的標準模板庫 (STL)。該版本的 STL 將被調整以與託管代碼和數據進行互動。那些習慣於使用 STL 編寫傳統 C++ 應用程式的程式員將發現，他們能應用相同的編碼技術來編寫基於 CLR 的應用程式。其次，新版本帶有增強安全功能的 C 運行庫 (CRT) 將隨 Visual C++ 2005 一起發佈。作為 Microsoft 對“可信賴計算”倡議的持續承諾的一部分，該版本的 CRT 將有助於增強集成了 CRT 的應用程式的集體安全性。

　　對於 Visual C++ 2005，開發人員將繼續選擇 Visual C++ 來產生最為苛求的應用程式和元件。Visual C++ 2005 透過提供令人興奮的新增和改進功能，為具有控制和性能意識的 C++ 開發人員擴展了在 Visual C++ .NET 2003 版中提供的核心功能集。
