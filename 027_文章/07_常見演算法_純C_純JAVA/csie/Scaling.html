Scaling

程度★★　難度★

將數值分為不同數量級，由最大的數量級開始，每個數量級分別計算一次答案，累計每次計算成果。

把一個問題的數值尺度縮小，由於數值尺度縮小的關係，問題就比較容易計算，也可以算得比較快。接著，計算出答案之後，再把問題和答案放大為原來大小。放大後的答案或許會有誤差，然而只要想辦法修正誤差，就能成為正確答案了！

目前Scaling應用較少，詳細運作過程請見「Single Source Shortest Paths: Scaling 」與「Maximum Flow: Capacity Scaling Algorithm」。

Randomized Algorithm

程度★★　難度★

當計算順序不影響正確答案的時候，就隨機安排計算順序，避免遇到最差情況，得到較佳的平均時間複雜度。

Fixed Parameter Algorithm

程度★★　難度★

當問題太過困難，就增加限制，將部分變數改成常數。

Distributed Algorithm

程度★★　難度★

當計算順序沒有前後相依的情況，就可以使用多台電腦同時計算。
